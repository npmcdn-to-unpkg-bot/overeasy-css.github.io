Configure

Compile

OverEasy is designed to be customized and modified as needed. It is written in Sass (SCSS) and compiled with Gulp so the CSS only contains the rule sets you actually need for your project. Any time you change anything in OverEasy, make sure you use Gulp to recompile.

To compile, open your terminal to the root folder of wherever you have OverEasy installed and simply run:

 gulp

OverEasy will recompile and export two new files in the /dist/css folder:

  • overeasy.css - Pretty version. For visual review.
  • overeasy.min.css - Minified and uglified. This is the one you actually serve to your site.

If you're gulp-savvy and want to mess with the compile options, you'll find gulpfile.js also in the root folder.

Settings

Most opinionated style properties are abstracted by variables all set together in /scss/index.scss

Variable names are semantic so their use should be pretty intuitive. You can also use variables as values for others. Here is a snippet as an example:

...

// *** Grid layout ***
$grid-columns: 12;
$breakpoint-sm: 40em;
$breakpoint-md: 52em;
$breakpoint-lg: 64em;
$container-width: $breakpoint-lg;

// *** Spacing ***
$space-1: 0.5rem;
$space-2: 1rem;
$space-3: 2rem;
$space-4: 4rem;

...

Look at the Settings section of /scss/index.scss for the full set of variables.

Modules

OverEasy's class declarations are separated out by functionality area. OverEasy is pretty lightweight, but you don't always need everything and why have a bigger file size than necessary. Comment out any module includes you don't need and recompile to slim down OverEasy to just the parts you want to use.

For example, let's say you want the color variants for text color, but not for background colors. You would just comment out the background color variants module like so:

...

// *** Typography ***
@import "./modules/typography/type-defaults";
@import "./modules/typography/type-utilities";
@import "./modules/typography/type-colors";
@import "./modules/typography/type-color-variants";

// *** Backgrounds ***
@import "./modules/backgrounds/backgrounds";
@import "./modules/backgrounds/background-colors";
// @import "./modules/backgrounds/background-color-variants";
@import "./modules/backgrounds/background-alpha";

...

Look at the Modules section of /scss/index.scss for the full set of modules.

The individual modules are located in /scss/modules if you want to see what any of them are doing. You can tinker with them if you want to, although if you're getting this deep into your config, you're probably better off just commenting out OverEasy's module and adding your own custom stylesheet to your project instead. OverEasy is atomic and modular and is designed to play well with any other custom styles or frameworks you employ.
